## Bind syntax

Every keybind follows this pattern:

```
BINDTYPE = MOD, KEY, DISPATCHER, [ARGS]
```

| Field | Description |
|-------|-------------|
| `BINDTYPE` | `bind`, `bindm`, `bindl`, `bindr`, `binde`, `bindel`, `bindtl`, etc. |
| `MOD` | Modifier keys separated by `_` (or empty for no modifier) |
| `KEY` | Key name or keycode (e.g. `Q`, `Return`, `XF86AudioRaiseVolume`) |
| `DISPATCHER` | Action to execute (see [Dispatchers](#dispatchers)) |
| `ARGS` | Optional arguments passed to the dispatcher |

```ini
$mainMod = SUPER

# Launch terminal
bind = $mainMod, Q, exec, kitty

# Close active window
bind = $mainMod, C, killactive,

# Move focus left
bind = $mainMod, left, movefocus, l
```

## Bind types

| Type | Description |
|------|-------------|
| `bind` | Standard bind — fires on key press |
| `bindm` | Mouse bind — fires on mouse button press with dragging |
| `bindl` | Locked bind — fires even when the session is locked |
| `bindr` | Release bind — fires on key release |
| `binde` | Repeat bind — continues firing while key is held |
| `bindel` | Repeat + locked — repeating bind that works on locked sessions |
| `bindtl` | Long-press bind — fires after the key is held for a duration |
| `bindn` | Non-consuming bind — event passes through to the application |
| `bindt` | Transparent bind — passes key to the focused window |
| `bindk` | Per-device bind — only fires for specified input devices |
| `bindi` | Ignore-mods bind — fires regardless of active modifiers |

Flags can be combined in a single keyword: `bindel`, `bindlr`, etc.

<CodeGroup>

```ini Mouse binds
# Move window by dragging
bindm = $mainMod, mouse:272, movewindow

# Resize window by dragging
bindm = $mainMod, mouse:273, resizewindow
```

```ini Locked/release binds
# Media keys — work even on lock screen
bindl = , XF86AudioNext, exec, playerctl next
bindl = , XF86AudioPause, exec, playerctl play-pause
bindl = , XF86AudioPlay, exec, playerctl play-pause
bindl = , XF86AudioPrev, exec, playerctl previous
```

```ini Repeat binds
# Volume — repeats while key is held
bindel = , XF86AudioRaiseVolume, exec, wpctl set-volume -l 1 @DEFAULT_AUDIO_SINK@ 5%+
bindel = , XF86AudioLowerVolume, exec, wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%-
```

</CodeGroup>

## Modifiers

Modifiers are specified in `UPPER_CASE` and combined with `_`:

| Modifier | Key |
|----------|-----|
| `SUPER` | Super / Meta / Windows key |
| `SHIFT` | Shift |
| `CTRL` or `CONTROL` | Control |
| `ALT` | Alt |
| `CAPS` | Caps Lock |
| `HYPER` | Hyper (rarely used physical key) |
| `MOD2` | Mod2 (Num Lock) |
| `MOD3` | Mod3 |
| `MOD5` | Mod5 |

Examples:

```ini
bind = SUPER, Q, exec, kitty              # Super only
bind = SUPER_SHIFT, Q, exec, kitty       # Super + Shift
bind = CTRL_ALT, Delete, exec, wlogout   # Ctrl + Alt
bind = , F1, exec, rofi -show run        # No modifier
```

## Special keys

Use XF86 key names for multimedia keys:

```ini
bindel = , XF86AudioRaiseVolume, exec, wpctl set-volume -l 1 @DEFAULT_AUDIO_SINK@ 5%+
bindel = , XF86AudioLowerVolume, exec, wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%-
bindel = , XF86AudioMute, exec, wpctl set-mute @DEFAULT_AUDIO_SINK@ toggle
bindel = , XF86AudioMicMute, exec, wpctl set-mute @DEFAULT_AUDIO_SOURCE@ toggle
bindel = , XF86MonBrightnessUp, exec, brightnessctl -e4 -n2 set 5%+
bindel = , XF86MonBrightnessDown, exec, brightnessctl -e4 -n2 set 5%-
```

Mouse buttons use `mouse:BUTTON_CODE`:

| Name | Code |
|------|------|
| Left button | `mouse:272` |
| Right button | `mouse:273` |
| Middle button | `mouse:274` |
| Scroll up | `mouse_up` |
| Scroll down | `mouse_down` |

```ini

# Relative workspaces
bind = $mainMod, mouse_down, workspace, e+1
bind = $mainMod, mouse_up, workspace, e-1

# Named workspace
bind = $mainMod, D, workspace, name:dev

bind = $mainMod, S, togglespecialworkspace, magic
bind = $mainMod SHIFT, S, movetoworkspace, special:magic
```

## Submaps

Submaps let you create modal keybind layers similar to Vim modes. Use `submap` to switch the active layer and `submap, reset` to return to normal.

```ini
# Enter resize mode
bind = $mainMod, R, submap, resize

submap = resize
    # Arrow keys resize the active window
    binde = , right, resizeactive, 10 0
    binde = , left, resizeactive, -10 0
    binde = , up, resizeactive, 0 -10
    binde = , down, resizeactive, 0 10

    # Escape exits the submap
    bind = , escape, submap, reset
submap = reset
```

<Note>
  All binds defined between `submap = NAME` and `submap = reset` belong to that submap. Binds outside any submap block belong to the default (global) layer.
</Note>

## Per-device binds

Target specific input devices using the `k` flag (`bindk`). The device list is passed as an extra argument after the modifier and key, using `[!]device1 device2 ...`. Prefix with `!` to match all devices *except* those listed.

```ini
# Only fire for a specific keyboard
bindk = , F1, my-keyboard-name, exec, notify-send hello

# Fire for all devices except the listed one
bindk = , F1, !my-keyboard-name, exec, notify-send hello
```

You can list device names with `hyprctl devices`.

## Unbinding

Remove an existing bind with `unbind`:

```ini
unbind = SUPER, Q
```

## Workspace switching examples

```ini
# Switch workspaces 1–10
bind = $mainMod, 1, workspace, 1
bind = $mainMod, 2, workspace, 2
bind = $mainMod, 3, workspace, 3
bind = $mainMod, 4, workspace, 4

bind = $mainMod, 6, workspace, 6
bind = $mainMod, 7, workspace, 7
bind = $mainMod, 8, workspace, 8
bind = $mainMod, 9, workspace, 9
bind = $mainMod, 0, workspace, 10

# Move window to workspace
bind = $mainMod SHIFT, 1, movetoworkspace, 1
bind = $mainMod SHIFT, 2, movetoworkspace, 2
# ...
```

## Config file location

Hyprland reads its configuration from `~/.config/hypr/hyprland.conf`. If the file does not exist on first launch, Hyprland generates a default one automatically.

<Warning>
  A generated default config starts with `autogenerated = 1`. Remove that line once you have edited the file, or Hyprland will display a warning on every startup.
</Warning>

## Splitting your config

Use the `source` keyword to include other files. This lets you organize your config into logical pieces:

```ini
# ~/.config/hypr/hyprland.conf
source = ~/.config/hypr/colors.conf
source = ~/.config/hypr/keybinds.conf
source = ~/.config/hypr/animations.conf
```

Glob patterns are also supported:

```ini
source = ~/.config/hypr/conf.d/*.conf
```

## Variable types

| Type | Description | Examples |
|------|-------------|---------|
| `bool` | Boolean flag | `true`, `false`, `yes`, `no`, `on`, `off`, `0`, `1` |
| `int` | Integer number | `0`, `10`, `-1` |
| `float` | Floating-point number | `1.0`, `0.5`, `2.5` |
| `color` | Color value | See below |
| `gradient` | Color gradient | See below |
| `string` | Text string | `"dwindle"`, `"us"` |
| `vec2` | 2D vector of floats | `0 0`, `1920 1080` |

### Colors

Colors are written as hexadecimal in one of two formats:

```ini
# RGB  — 0xRRGGBB
col.inactive_border = 0xff595959

# RGBA — rgba(RRGGBBAA)
col.active_border = rgba(33ccffee)
```

### Gradients

Gradient values accept multiple colors and an optional angle:

```ini
# Two-color gradient at 45 degrees
col.active_border = rgba(33ccffee) rgba(00ff99ee) 45deg

# Single color (no gradient)
col.inactive_border = rgba(595959aa)
```

Up to 10 colors are supported per gradient. The angle is specified in degrees.

## Environment variables

Set environment variables that Hyprland will export to all child processes:

```ini
env = XCURSOR_SIZE,24
env = HYPRCURSOR_SIZE,24
env = QT_QPA_PLATFORM,wayland
```

The syntax is `env = KEY,VALUE`. These are applied before any autostart programs run.

<Accordion title="general">

Controls layout, borders, gaps, and core window behaviour.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `border_size` | int | `1` | Width of window borders in pixels |
| `gaps_in` | gap | `5` | Gaps between windows |
| `gaps_out` | gap | `20` | Gaps between windows and monitor edges |
| `float_gaps` | gap | `0` | Gaps around floating windows |
| `gaps_workspaces` | int | `0` | Gaps between workspaces |
| `col.active_border` | gradient | `0xffffffff` | Border color of the focused window |
| `col.inactive_border` | gradient | `0xff444444` | Border color of unfocused windows |
| `col.nogroup_border` | gradient | `0xffffaaff` | Border color for windows not in a group |
| `col.nogroup_border_active` | gradient | `0xffff00ff` | Border color for active window not in a group |
| `resize_on_border` | bool | `false` | Allow resizing by dragging window borders |
| `extend_border_grab_area` | int | `15` | Extra grab area beyond the border in pixels |
| `hover_icon_on_border` | bool | `true` | Change cursor when hovering over a resizable border |
| `allow_tearing` | bool | `false` | Enable tearing for reduced input latency (see Tearing) |
| `resize_corner` | int | `0` | Force a specific corner for resizing (0 = any) |
| `layout` | string | `dwindle` | Active layout: `dwindle`, `master`, or `scrolling` |
| `no_focus_fallback` | bool | `false` | Disable focus fallback to the next available window |

```ini
general {
    gaps_in = 5
    gaps_out = 20
    border_size = 2
    col.active_border = rgba(33ccffee) rgba(00ff99ee) 45deg
    col.inactive_border = rgba(595959aa)
    resize_on_border = false
    allow_tearing = false
    layout = dwindle
}
```

<Accordion title="decoration">

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `rounding` | int | `0` | Corner rounding radius in pixels |

| `dim_around` | float | `0.4` | Dim strength for `dimaround` window rule |
| `screen_shader` | string | `` | Path to a GLSL screen shader |
| `border_part_of_window` | bool | `true` | Treat border area as part of the window for clicks |

**Shadow sub-section:**

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `shadow:enabled` | bool | `true` | Enable window drop shadows |
| `shadow:range` | int | `4` | Shadow reach in pixels |
| `shadow:render_power` | int | `3` | Shadow quality (1–4) |
| `shadow:color` | color | `0xee1a1a1a` | Shadow color |

| `shadow:offset` | vec2 | `0 0` | Shadow offset |
| `shadow:scale` | float | `1.0` | Shadow texture scale |
| `shadow:sharp` | bool | `false` | Render a sharp (non-blurred) shadow |

**Blur sub-section:**

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `blur:enabled` | bool | `true` | Enable background blur |
| `blur:size` | int | `8` | Blur kernel size |
| `blur:passes` | int | `1` | Number of blur passes (higher = more blur) |
| `blur:ignore_opacity` | bool | `true` | Blur ignores window opacity |
| `blur:new_optimizations` | bool | `true` | Enable blur rendering optimizations |
| `blur:xray` | bool | `false` | Blur sees through windows to the wallpaper |
| `blur:noise` | float | `0.0117` | Noise amount added to blur |
| `blur:contrast` | float | `0.8916` | Contrast boost for blurred areas |
| `blur:brightness` | float | `1.0` | Brightness boost for blurred areas |
| `blur:vibrancy` | float | `0.1696` | Color vibrancy boost |
| `blur:special` | bool | `false` | Blur the special workspace background |
| `blur:popups` | bool | `false` | Blur popup/tooltip windows |

```ini
decoration {
    rounding = 10
    rounding_power = 2

    active_opacity = 1.0
    inactive_opacity = 1.0

    shadow {
        enabled = true
        range = 4
        render_power = 3
        color = rgba(1a1a1aee)
    }

    blur {
        enabled = true
        size = 3
        passes = 1
        vibrancy = 0.1696
    }
}
```

## Per-device configuration

Use a `device {}` block to override settings for a specific input device. The `name` field must match the device name shown by `hyprctl devices`.

```ini
device {
    name = epic-mouse-v1
    sensitivity = -0.5
}
```

Any `input {}` option can be overridden inside a `device {}` block.

## Variables ($)

You can define reusable variables with `$`:

```ini
$mainMod = SUPER
$terminal = kitty

bind = $mainMod, Q, exec, $terminal
```

Variable names are case-sensitive and can contain letters, digits, and underscores.
